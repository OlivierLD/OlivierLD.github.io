Building the Speech Service

  • Setup an OCI Compute Instance as described here, use compartment "development".
  • On that instance, upgrade the git version.
    See instructions here.
    On the instance: 
    $ sudo su
$ yum install curl-devel expat-devel gettext-devel openssl-devel zlib-devel -y
$ yum install gcc perl-ExtUtils-MakeMaker -y
    To remove existing version of git: 
    $ yum remove git -y
    Option 1:
    Download the archive from your laptop, and send it to your instance:
    From laptop (Like Mac) 
    $ export http_proxy=http://www-proxy.us.oracle.com:80
$ wget -e use_proxy=yes -e http_proxy=$http_proxy -e https_proxy=$http_proxy https://www.kernel.org/pub/software/scm/git/git-2.30.0.tar.gz
    Send it to the Compute Instance 
    $ scp -i ~/.ssh/id_rsa_00 ./git-2.30.0.tar.gz  opc@100.111.136.104:~
    On the instance: 
    $ sudo mv ~/git-2.30.0.tar.gz /usr/src
$ cd /usr/src
$ sudo su
$ tar xzf git-2.30.0.tar.gz
$ make prefix=/usr/local/git all
$ make prefix=/usr/local/git install
$ echo "export PATH=/usr/local/git/bin:$PATH" >> /etc/bashrc 
$ exit
$ source /etc/bashrc
$ git --version
git version 2.30.0
$
    Before starting the build, do a git clean -fdx.
    Option 2:
    On the instance: 
    $ sudo yum install -y curl-devel expat-devel gettext-devel openssl-devel zlib-devel gcc perl-ExtUtils-MakeMaker

$ sudo yum remove git

$ tar xzf git-2.21.0.tar.gz
$ cd git-2.21.0
$ make prefix=/usr/local all
$ sudo make prefix=/usr/local install

    Then

    Make sure docker-compose is in your PATH 
    $ sudo su -
$ yum remove docker-compose
$ source /home/opc/.proxies.sh
# export http_proxy=http://www-proxy-hqdc.us.oracle.com:80
# export https_proxy=http://www-proxy-hqdc.us.oracle.com:80
$ curl -L "https://github.com/docker/compose/releases/download/1.24.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
$ chmod 755 /usr/local/bin/docker-compose
  • On that instance, clone the repo (master branch)

  • Important:

    If you have other service(s) running on your instance (like ODA bots), do reboot your instance!!
  • Start the build:
    Make sure you've set the right System variables: 
    export ORACLE_ID=akeu.coucou@oracle.com
export JAVA_HOME=/usr/java/jdk-11.0.7

    $ ./buildDeploy.sh -x tests
  • Note:
    To have the project compiled locally in IntelliJ running on your laptop, you need to set the Gradle flag skip.requirement to true.
    To do so, add the following line to your local ~/.gradle/gradle.properties: 
    skip.requirements=true
    You can also start the build from the command line: 
    $ ./buildDeploy.sh -x test -x lint -Pskip.requirements=true

Access the Speech Service

  • The README.md at https://bitbucket.oci.oraclecorp.com/projects/ODA/repos/bots-speech/browse will tell you what to do next.
  • Use the Web SDK, from your browser.
    From Google Chrome, go to chrome://flags to change the configuration,
    • enabled Insecure origins treated as secure
    • add http://<ip>
    Then reach http://100.111.136.104/test/dev-latest, where 100.111.136.104 is the IP Address of your instance.
    Note: The WebSDK is using Node-RED, you can reach the Node-RED UI at http://100.111.136.104:1881/ where again, 100.111.136.104 is the IP Address of your instance.
  • Use the Speech Service from curl:
    Have a shell script (named for example curl.speech.sh) like this: 
    #!/bin/bash
MACHINE_IP=100.111.136.104
# MACHINE_IP=147.154.101.151  # speech.aiservice.us-phoenix-1.oci.oraclecloud.com
TENANT_ID=odaserviceinstance00
OPTION=generic
# OPTION=alarm
AUDIO_FILE=./foo.wav
#
if [[ "$1" != "" ]]; then
    AUDIO_FILE=$1
    echo -e "{ \"message\": \"Will transcript user-provided ${AUDIO_FILE}\" }"
else  
    echo -e "{ \"message\": \"Will transcript default ${AUDIO_FILE}\" }"
fi
#
# "http://${MACHINE_IP}/voice/recognize/en-us/generic"
# Convert m4a to wav on line: https://www.zamzar.com/
#
curl -X POST \
    --data-binary @${AUDIO_FILE} \
    -H "Bots-TenantId: ${TENANT_ID}" \
    -H "channelId: recognize" \
    -H "Content-Type: audio/wav" \
    "http://${MACHINE_IP}/voice/recognize/en-us/${OPTION}"
    The default file to process is foo.wav, run the script with jq like this:
