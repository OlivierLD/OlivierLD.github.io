Doc pour le web développeur

Sans surprise, la racine du site est index.html.

Pour le moment (Dec 2023), il n'y a dans le code AUCUNE dépendance vis-à-vis d'un framework (pas de WordPress, JQuery, ReactJS, Underscore, Knockout) externe.

Le site est une SPA (Single Page Application). Ça permet de réduire la taille des ressources de manière substancielle.
Autres fichiers reliés à l'index :
  • passe-coque.css
  • passe-coque.js
  • passe-coque.menu.css

Deux menus sont disponibles :
  • Un menu hiérarchique et déroulant
  • Une barre latérale de navigation, activée avec un hamburger (≡)
L'un est destiné aux cell-phones (la barre latérale), l'autre aux autres périphériques.
La distinction est faite dans un code en JavaScript, dans le window.onload, dans le fichier index.html, en fonction des valeurs de window.innerWidth et window.innerHeight.
La distinction est faite dans passe-coque.css, en fonction de la largeur de l'écran.

Les menus (les deux) sont composés de links, distingués par leurs id.
Les actions à prendre sur un click sont prises par la fonction clack, en fonction des ids et de la langue courante (FR ou EN). La fonction clack, définie elle aussi dans passe-coque.js.
Si l'id a une valeur de _21 et la langue est FR, alors le contenu d'un fichier 21_FR.html est lu dynamiquement, et affiché dans la <div id="current-content">.
Warning!: des exceptions à cette règle existent. Voir le code de la fonction clack, dans le fichier passe-coque.js.

Chargement du site :
Voir dans index.html, le code (javascript) du window.onload y charge (dynamiquement) 1_FR.html.
On peut changer la langue par defaut au chargement avec un QS param lang. La langue par defaut est FR. On peut charger:
http://passe-coque.com/?lang=EN.
Les images du haut de la home page ont un rapport de 3.5:1. Au format jpg, une dimension de 1500 x 429 semble être la bonne.
La traduction (des menus) est assurrée par la fonction updateMenu, définie dans passe-coque.js.
Tout le site se comporte comme une SPA (Single Page Application). La page index.html reste toutjours affichée, le contenu de la div "current-content" est changé dynamiquement en fonction des choix de l'utilisateur. Encore une fois, voir à ce sujet le code de la fonction clack, dans le fichier passe-coque.js.
Les photos des membres du bureau ont un rapport de 1:1 (genre 400 x 400).
La flotte : La liste des bateaux est un JSON Object THE_FLEET dans passe-coque.js.
Les fonctions updateFilter et fillOutFleet etablissent la liste des bateaux, en fonction de la valeur des filtres (radio-boutons).
Les images de bateau ont une taille de 376 × 376, rapport 1:1, obviously.
Les actu sont gérées par le JSON Array INFO_SECTION dans passe-coque.js, et la fonction fillOutActu.

La version par défaut est la version française. La traduction - éventuelle - est assurée dynamiquement par la fonction translate (dans passe-coque.js). C'est un peu compliqué, mais ça apporte plus de souplesse (quand il n'est pas nécessaire de traduire, par exemple).
